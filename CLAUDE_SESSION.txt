  ---
  1. Implementation Plan

  Phase 1 — Data layer

  - Define TypeScript types for the raw JSON schema (ActivityNode, ResourceNode,
   Dependency)
  - Write a transformer that converts the raw JSON into React Flow Node[] and
  Edge[] arrays
  - Apply an automatic layout algorithm (dagre or a simple tiered layout) to
  assign initial positions — activities on the left column, resources on the
  right

  Phase 2 — Graph canvas

  - Integrate @xyflow/react (ReactFlow) with the transformed nodes/edges
  - Enable built-in controls (zoom, pan, fit-view)
  - Display a minimap for orientation

  Phase 3 — Custom node types

  - ActivityNode — pill/card shape, coloured by priority (critical = red, high =
   orange, medium = blue), shows name + RTO badge
  - ResourceNode — distinct shape/colour per resource_type (technology, people,
  third_party, building, equipment), shows name + type badge

  Phase 4 — Selection & detail panel

  - Clicking a node opens a side panel with all its metadata (owner/contact,
  vendor if applicable, RTO, priority, connected nodes)
  - Clicking an edge highlights that connection
  - Clicking the canvas background clears selection

  Phase 5 — Filtering / legend

  - A legend explaining node colours/shapes
  - Optional: toggle-filter by node type or priority (if time allows)

  Implication: These nodes will be highly connected — layout should avoid edge

  ────────────────────────────────────────
